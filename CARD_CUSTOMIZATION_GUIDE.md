# 🎴 卡片自定義指南

## 概述

現在所有的卡片內容都通過 `cards-config.json` 文件來配置！你可以輕鬆地：
- 修改卡片的標題、引言、獎勵內容
- 替換卡片圖片（只需把圖片放到 `assets/cards/` 文件夾）
- 調整稀有度配色（進階功能）

---

## 📁 文件結構

```
pong_val/
├── cards-config.json        # 卡片配置文件（你主要編輯這個！）
├── index.html               # 遊戲主文件（不需要改）
└── assets/
    └── cards/
        ├── example.svg      # 佔位圖（所有卡片暫時使用這個）
        ├── R1.jpg          # 你的圖片 - R卡 #1
        ├── SR1.jpg         # 你的圖片 - SR卡 #1
        └── UR1.jpg         # 你的圖片 - UR卡（最重要！）
```

## 🎨 如何自定義卡片

### 步驟 1: 編輯 `cards-config.json`

打開 `cards-config.json`，你會看到這樣的結構：

```json
{
  "cards": [
    {
      "id": "R1",
      "rarity": "R",
      "name": "初次相遇",
      "title": "「圖書館的那個午後」",
      "quote": "你問我借了一支筆，從此我的心就借給了你。",
      "image": "assets/cards/example.svg",
      "perk": {
        "label": "愛的憑證",
        "value": "一個擁抱 (隨時兌換)"
      }
    },
    // ... 更多卡片
  ]
}
```

### 各欄位說明

| 欄位 | 說明 | 範例 |
|------|------|------|
| `id` | 卡片唯一識別碼 | `"R1"`, `"SR2"`, `"UR1"` |
| `rarity` | 稀有度 | `"R"`, `"SR"`, `"SSR"`, `"UR"` |
| `name` | 卡片名稱（簡短） | `"初次相遇"` |
| `title` | 卡片標題（顯示在卡片上） | `"「圖書館的那個午後」"` |
| `quote` | 引言/描述（顯示在卡片上） | `"你問我借了一支筆，從此..."` |
| `image` | 圖片路徑 | `"assets/cards/R1.jpg"` |
| `perk.label` | 獎勵標籤 | `"愛的憑證"` |
| `perk.value` | 獎勵內容 | `"一個擁抱 (隨時兌換)"` |

## 🖼️ 如何替換圖片

### 方法 1: 逐張替換（推薦）

1. 準備你的圖片（建議尺寸：400×600 px）
2. 把圖片命名為對應的ID，例如：
   - R1 卡片 → `R1.jpg` 或 `R1.png`
   - SR2 卡片 → `SR2.jpg` 或 `SR2.png`
   - UR1 卡片 → `UR1.jpg` 或 `UR1.png`
3. 把圖片放到 `assets/cards/` 文件夾
4. 編輯 `cards-config.json`，修改對應卡片的 `image` 欄位：

```json
{
  "id": "R1",
  "image": "assets/cards/R1.jpg",  // 改成你的圖片路徑
  // ... 其他欄位
}
```

### 方法 2: 使用相同的圖片

如果你想讓某些卡片使用同一張圖片（例如暫時用佔位圖）：

```json
{
  "id": "R1",
  "image": "assets/cards/example.svg",  // 多張卡片可以用同一張圖
  // ...
}
```

### 圖片規格建議

| 項目 | 建議值 | 備註 |
|------|--------|------|
| **尺寸** | 400×600 px（2:3比例） | 可以更大，但保持比例 |
| **格式** | JPG, PNG, SVG | PNG支持透明背景 |
| **文件大小** | < 500 KB | 太大會影響加載速度 |
| **裁切** | **object-fit: cover** | 圖片會自動裁切到中間，不會變形 |

**重要：** 圖片會使用 `object-fit: cover` 來填滿卡片，所以：
- 圖片會裁切到中間部分（center crop）
- 不會拉伸變形
- 確保重要內容在圖片中央

## 🎭 進階：調整稀有度配色

如果你想修改不同稀有度的顏色和光暈效果，可以編輯 `rarityConfig` 部分：

```json
"rarityConfig": {
  "SSR": {
    "color": "#9400D3",                              // 文字顏色
    "glowColor": "rgba(148, 0, 211, 0.8)",          // 光暈顏色
    "name": "史詩",                                  // 稀有度名稱
    "bgGradient": "linear-gradient(135deg, #f093fb 0%, #9400D3 100%)"  // 背景漸層
  }
}
```

**一般不需要修改這部分**，除非你想完全自定義配色方案。

## 🚀 快速開始流程

### 最簡單的方式（只改文字）

1. 打開 `cards-config.json`
2. 找到你想改的卡片
3. 修改 `title`, `quote`, `perk.value` 這三個欄位
4. 保存文件
5. 刷新瀏覽器 - 完成！

### 完整自定義（文字+圖片）

1. **準備圖片**
   - 10張照片，命名為 R1.jpg, R2.jpg, ..., UR1.jpg
   - 放到 `assets/cards/` 文件夾

2. **編輯配置**
   - 打開 `cards-config.json`
   - 逐張修改 `title`, `quote`, `image`, `perk`

3. **測試**
   - 刷新瀏覽器
   - 玩遊戲賺Token
   - 抽卡看效果

4. **調整**
   - 如果有錯誤，檢查 JSON 格式是否正確（逗號、引號）
   - 確認圖片路徑正確

## ⚠️ 常見問題

### Q: 改完配置後遊戲沒有變化？
A:
1. 清除瀏覽器緩存（Cmd+Shift+R 或 Ctrl+Shift+R）
2. 檢查 JSON 格式是否正確（可以用 [JSONLint](https://jsonlint.com/) 驗證）
3. 打開瀏覽器控制台（F12）看有沒有錯誤訊息

### Q: 圖片沒有顯示？
A:
1. 確認圖片路徑正確（區分大小寫！）
2. 確認圖片在 `assets/cards/` 文件夾內
3. 確認圖片格式是 JPG/PNG/SVG
4. 如果圖片沒有，會顯示彩色佔位圖（這是正常的）

### Q: 如何只更新部分卡片？
A: 沒問題！只修改你想改的卡片，其他保持 `example.svg` 即可。

### Q: 可以添加更多卡片嗎？
A: 可以！在 `cards` 陣列中添加新的卡片對象，記得：
- 設定唯一的 `id`
- 選擇適當的 `rarity` (R/SR/SSR/UR)
- 準備對應的圖片

### Q: JSON 格式錯誤怎麼辦？
A: 常見錯誤：
- 漏了逗號 `,`
- 多餘的逗號（最後一個元素後面不能有逗號）
- 引號不匹配
- 使用中文引號「」而不是英文引號 ""

## 💡 小提示

1. **先修改文字，再替換圖片** - 這樣可以先看到效果
2. **UR卡最重要** - 優先準備 UR1 的圖片和內容
3. **保持備份** - 修改前複製一份 `cards-config.json`
4. **測試每次修改** - 改一點就測試一次，不要一次改太多
5. **使用相對路徑** - 圖片路徑用 `assets/cards/xxx.jpg`，不要用絕對路徑

## 🎨 圖片處理建議

### 推薦工具：
- **Photoshop / GIMP** - 調整尺寸和裁切
- **Canva** - 在線設計工具
- **手機照片編輯器** - iOS 照片 / Google Photos

### 處理步驟：
1. 選一張你喜歡的照片
2. 裁切成 2:3 比例（例如 400×600）
3. 確保主題在中央
4. 導出為 JPG（品質 80-90%）
5. 檢查文件大小 < 500 KB
